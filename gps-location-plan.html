1
Frontend – Nút "Dùng vị trí GPS"
Không cần đụng backend, không cần API key
Dễ · ~2h
  • 🖱️
    Thêm nút "Dùng vị trí của tôi" vào Location.jsx
    Hiển thị nút với icon GPS bên cạnh ô search. Khi click → gọi navigator.geolocation.getCurrentPosition(). Xử lý 3 trạng thái: loading / success / permission denied.
    Location/Location.jsx
  • 🔄
    Reverse geocode bằng Nominatim (OpenStreetMap) – miễn phí
    Gọi https://nominatim.openstreetmap.org/reverse?lat=&lon=&format=json. Extract ra suburb hoặc city_district → đây là tên quận. Không cần API key, không tốn tiền.
    hook/useGPSLocation.jsx (mới)
  • 🗂️
    Match tên quận với VietnamCity.json → lấy pincode
    Dùng lại hàm removeAccents() đã có trong useVietnamCity.jsx để fuzzy match không dấu. Nếu match được → gọi updateLocation() như thường. Nếu không match → fallback báo lỗi.
    hook/useGPSLocation.jsx
2
Database – Thêm tọa độ lat/lng cho nhà hàng
Prerequisite để sort theo khoảng cách thật sự
Trung bình · ~3h
  • 📊
    Thêm field lat và lng vào collection restaurant
    Hiện tại restaurant chỉ có pincode (dạng string). Cần thêm 2 field số vào MongoDB. Viết migration script chạy 1 lần để geocode từ địa chỉ hiện có.
    Database/restaurant schema
  • 🗺️
    Tạo migration script geocode địa chỉ nhà hàng
    Script Node.js: đọc từng restaurant → gọi Nominatim với address field → lưu lat/lng vào MongoDB. Chạy 1 lần, rate limit 1 req/s theo ToS Nominatim.
    Backend/src/scripts/migrateLatLng.js (mới)
  • 🏪
    Form thêm nhà hàng mới – bổ sung lat/lng tự động
    Trong useCreateNewRestaurant.jsx: khi seller submit địa chỉ, tự động geocode lấy lat/lng trước khi gửi lên backend.
    hook/useCreateNewRestaurant.jsx
3
Backend – Sort by distance (Haversine)
Thay pincode-filter bằng distance-sort thực sự
Trung bình · ~3h
  • 📐
    Thêm endpoint GET /nearbyrestaurants?lat=&lng=&radius=
    Thay vì filter theo pincode, lấy tất cả nhà hàng có lat/lng → tính khoảng cách Haversine → lọc trong bán kính (mặc định 10km) → sort gần nhất lên đầu.
    routes/NearbyRestaurants/ (mới)
  • 🔢
    Hàm Haversine tính khoảng cách
    Công thức chuẩn tính km giữa 2 tọa độ. Thêm field distanceKm vào response để frontend hiển thị "2.3 km". Không cần thư viện ngoài.
    db/NearbyRestaurants/getNearbyRestaurants.js (mới)
  • 🔀
    Thêm sorting option "nearest" vào endpoint sorting hiện có
    Extend /availablerestaurants/:pincode/:sorting để nhận thêm lat và lng query param khi sorting = "nearest". Backward-compatible với các sort cũ.
    db/AvailableRestaurantsSorting/getAvailableRestaurantsSorting.js
4
Frontend – Hiển thị distance + UX polish
User thấy kết quả rõ ràng và trực quan
Dễ · ~2h
  • 📍
    Hiển thị badge "2.3 km" trên card nhà hàng
    Dùng field distanceKm từ response. Hiển thị bên cạnh thời gian giao hàng. Nếu user chưa set GPS thì ẩn badge.
    restaurants/RestaurantCard.jsx
  • 🔃
    Thêm chip "Gần nhất" vào sort bar
    Thêm option "Gần nhất" vào danh sách sort hiện có (Giá thấp, Giá cao, Rating, Thời gian…). Chỉ active nếu user đã set GPS. Nếu chưa → click sẽ trigger flow xin quyền GPS.
    home/SortingChips.jsx
  • 💾
    Lưu lat/lng user vào localStorage (không chỉ pincode)
    Bổ sung updateGPSCoords(lat, lng) và fetchGPSCoords() vào LocationLocalStorage.jsx. Hook useAvailableRestaurantsSorting đọc thêm coords khi sorting = "nearest".
    hook/LocationLocalStorage.jsx
